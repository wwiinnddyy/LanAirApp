# 插件收录标准

本文档定义了 LanAirApp 官方市场收录插件的审核标准。所有提交收录申请的插件都需满足以下条件。

## 一、必要条件（必须满足）

以下条件任一不满足，收录申请将被直接拒绝。

### 1.1 API 版本合规

- `plugin.json` 中 `apiVersion` 必须为 `4.x`（当前为 `4.0.0`）
- 插件入口使用 `Initialize(HostBuilderContext, IServiceCollection)` 签名
- 组件注册使用 `PluginDesktopComponentOptions`（而非旧版 API）

### 1.2 包格式正确

- `.laapp` 包为合法的 ZIP 压缩包
- 包内包含 `plugin.json`，且 JSON 格式合法
- `plugin.json` 中 `id`、`name`、`author`、`version`、`apiVersion`、`entranceAssembly` 字段非空
- `entranceAssembly` 指向的程序集在包内存在
- 包内不包含无关的开发产物（`.pdb`、`obj/`、`.vs/` 等）

### 1.3 GitHub Release 可达

- 插件仓库为 `github.com` 上的公开仓库
- 最新 Release 中包含 `.laapp` 后缀的 Asset
- Release Tag 格式为 `v<版本号>`（如 `v1.0.0`）

### 1.4 本地化

- 提供 `Localization/zh-CN.json`（中文）
- 提供 `Localization/en-US.json`（英文）
- 本地化文件 JSON 格式合法
- 关键 UI 文本均有对应的本地化 key

### 1.5 基本功能可用

- 插件能在 LanMountainDesktop 中正常加载
- 插件不会导致宿主崩溃或严重性能问题
- 桌面组件（如有）能正常显示
- 设置页（如有）能正常打开和操作

## 二、推荐条件（强烈建议满足）

以下条件不满足不会直接拒绝，但会影响审核优先级和通过概率。

### 2.1 主题适配

- 亮色主题下显示正常
- 暗色主题下显示正常
- 主题切换时无视觉异常

### 2.2 外观系统

- 使用 `IPluginAppearanceContext` / `PluginAppearanceSnapshot` 消费外观信息
- 使用 `PluginCornerRadiusPreset` 处理圆角（而非硬编码）
- 组件尺寸自适应（响应 CellSize 和 Bounds 变化）

### 2.3 市场元数据

- Release 中包含 `market-manifest.json` Asset
- `market-manifest.json` 的 `schemaVersion` 为 `2.0.0`
- 插件图标清晰且与功能相关

### 2.4 代码质量

- 代码结构清晰，职责分离
- 无明显的内存泄漏（如未释放的 Timer、未取消的事件订阅）
- 异步操作使用 `async/await`，不使用 `.Result` 或 `.Wait()`
- 线程安全（共享状态使用锁或并发集合）

### 2.5 文档

- `README.md` 包含插件功能说明
- `README.md` 包含安装和使用说明
- `README.md` 包含开发/构建说明（如适用）

## 三、拒绝条件（满足任一即拒绝）

### 3.1 安全问题

- 包含恶意代码（后门、挖矿、数据窃取等）
- 未经用户同意访问敏感数据（文件系统、网络、注册表等）
- 使用不安全的网络通信（HTTP 而非 HTTPS，不验证证书等）
- 包含硬编码的密钥、令牌或凭据

### 3.2 稳定性问题

- 加载后导致宿主崩溃
- 导致宿主内存持续增长（内存泄漏）
- 导致宿主 CPU 占用异常
- 阻塞宿主 UI 线程

### 3.3 合规问题

- `apiVersion` 不为 `4.x`
- `plugin.json` 格式不合法或字段缺失
- `.laapp` 包格式不正确
- 仓库不是 `github.com` 上的公开仓库
- 插件 ID 与已收录插件冲突
- 插件 ID 命名不规范（如包含空格、特殊字符）

### 3.4 质量问题

- 插件功能极不完整，无法实际使用
- UI 严重错乱或不可交互
- 本地化文本大量缺失

## 四、审核流程

### 4.1 自动校验（CI）

提交 PR 后，CI 会自动执行以下校验：

1. **index.json 格式校验**：Validator 校验索引的 JSON Schema 合法性
2. **V3 索引结构校验**：
   - `schemaVersion` 为 `3.0.0`
   - `sourceId`、`sourceName` 非空
   - 每个 `pluginId` 非空且无重复
   - `repositoryUrl` 为有效 HTTP(S) URL
   - `packageSources` 恰好 3 个，按 `releaseAsset` → `rawFallback` → `workspaceLocal` 顺序

### 4.2 人工审核

维护者会进行以下审核：

1. **注册表条目审核**：字段是否完整、URL 是否可达、标签是否合理
2. **插件功能审核**：下载 `.laapp` 并在 LanMountainDesktop 中实际加载测试
3. **代码审核**：检查插件源码的安全性、代码质量和合规性
4. **UI 审核**：检查桌面组件和设置页的显示效果

### 4.3 审核结果

| 结果 | 说明 |
|------|------|
| **通过** | 满足所有必要条件，推荐条件基本满足，合并 PR |
| **需修改** | 满足必要条件但推荐条件有缺失，维护者留言说明需修改的内容 |
| **拒绝** | 不满足必要条件或触发拒绝条件，关闭 PR 并说明原因 |

## 五、已收录插件的维护

### 5.1 版本更新

已收录插件发布新版本时，无需重新提交收录申请。IndexBuilder 会自动从 GitHub API 获取最新 Release。

### 5.2 下架条件

已收录插件如出现以下情况，可能被从市场中移除：

- 长期无法加载或运行（与宿主新版本不兼容）
- 发现安全问题
- 仓库被删除或转为私有
- 作者申请下架

### 5.3 信息更新

如需更新已收录插件的注册信息（标签、能力提示、图标等），提交 PR 修改 `official-plugins.json` 中对应条目即可。

## 六、插件 ID 命名规范

插件 ID 是全局唯一标识，建议遵循以下规范：

- 格式：`LanMountainDesktop.<PluginName>` 或 `<Brand>.<PluginName>`
- 使用 PascalCase（大驼峰）
- 不包含空格、连字符或特殊字符
- 与 `plugin.json` 中 `id` 和程序集命名保持一致

**示例**：
- `LanMountainDesktop.SamplePlugin` — 官方示例
- `VoiceHubLanDesktop` — VoiceHub 品牌插件
- `Elysia.LanDesktopConnect` — Elysia 品牌插件
