# 插件收录指南

本文档详细说明如何将插件收录到 LanAirApp 官方市场。如果你是插件作者，请按照以下步骤操作。

## 收录流程总览

```
开发插件 → 打包 .laapp → 发布 Release → 提交收录申请 → 审核通过 → 自动生效
```

## 第一步：确认前置条件

在提交收录申请之前，请确保满足以下条件：

| 条件 | 说明 |
|------|------|
| `apiVersion` 为 `4.x` | `plugin.json` 中 `apiVersion` 必须为 `4.0.0` 或更高 4.x 版本 |
| GitHub Release 已创建 | 在插件仓库中创建 Release，并上传 `.laapp` 文件作为 Asset |
| `.laapp` 包格式正确 | 包内包含合法的 `plugin.json` 和 `entranceAssembly` 指向的程序集 |
| 本地化文件完整 | 提供 `Localization/zh-CN.json` 和 `Localization/en-US.json` |
| README.md 存在 | 插件仓库根目录包含 `README.md` |
| 插件可正常运行 | 在 LanMountainDesktop 中实际加载并运行正常 |

## 第二步：打包插件

使用官方打包工具生成 `.laapp` 文件：

```powershell
dotnet run --project <PluginPackager路径> -- --input <构建输出目录> --output <插件ID>.<版本号>.laapp --overwrite
```

### .laapp 包应包含的内容

- `plugin.json` — 插件清单
- 插件主程序集 — 与 `entranceAssembly` 对应的 DLL
- 依赖程序集和 `.deps.json`
- `Localization/zh-CN.json` 和 `Localization/en-US.json`
- 共享契约 DLL（如有 `sharedContracts`）
- 其他运行时资源文件

> **注意**：不要将无关的开发产物（如 `.pdb`、`obj/` 目录等）打入包内。

## 第三步：发布 Release

在插件 GitHub 仓库中创建 Release：

1. **Tag 格式**：`v<版本号>`，如 `v1.0.0`
2. **上传 Asset**：
   - **必须**：`<插件ID>.<版本号>.laapp`
   - **推荐**：`market-manifest.json`（提供额外的市场元数据）
   - **推荐**：`sha256.txt` 和 `md5.txt`（校验和文件）

### market-manifest.json 示例

```json
{
  "schemaVersion": "2.0.0",
  "pluginId": "LanMountainDesktop.YourPlugin",
  "displayName": "Your Plugin",
  "description": "插件功能描述",
  "iconAsset": "icon.png",
  "tags": ["tag1", "tag2"],
  "compatibility": {
    "pluginApiVersion": "4.0.0",
    "minHostVersion": "0.7.4"
  }
}
```

## 第四步：提交收录申请

有两种方式：

### 方式 A：提交 Pull Request（推荐）

1. Fork [LanAirApp](https://github.com/wwiinnddyy/LanAirApp) 仓库
2. 编辑 `airappmarket/registry/official-plugins.json`，在 `plugins` 数组末尾添加新条目
3. 提交 PR，使用 **"插件收录申请"** 模板填写信息

### 方式 B：提交 Issue

在 [LanAirApp Issues](https://github.com/wwiinnddyy/LanAirApp/issues) 中使用 **"插件收录申请"** 模板提交 Issue，维护者会代为编辑注册表。

## 第五步：等待审核

- CI 会自动校验 `index.json` 的合法性
- 维护者会审核插件的功能、代码质量和合规性
- 如有问题，维护者会在 PR/Issue 中留言

## 第六步：自动生效

审核通过并合并后：

1. CI 自动重新生成 `airappmarket/index.json`
2. 新插件即刻出现在官方市场中
3. LanMountainDesktop 宿主下次刷新市场时即可发现新插件

## 注册表条目字段说明

`airappmarket/registry/official-plugins.json` 中每个插件条目的字段说明：

### 必填字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | string | 插件唯一标识，与 `plugin.json` 中 `id` 一致 | `"LanMountainDesktop.YourPlugin"` |
| `repositoryUrl` | string | GitHub 仓库 URL（**必须为 github.com**） | `"https://github.com/owner/repo"` |

### 推荐字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `marketManifestAssetName` | string | Release 中市场清单 Asset 的名称，默认 `"market-manifest.json"` | `"market-manifest.json"` |
| `projectUrl` | string | 项目主页 URL | `"https://github.com/owner/repo"` |
| `readmeUrl` | string | README 文件的 raw URL | `"https://raw.githubusercontent.com/owner/repo/main/README.md"` |
| `homepageUrl` | string | 主页 URL | `"https://github.com/owner/repo"` |
| `iconUrl` | string | 插件图标 URL（支持 SVG/PNG） | `"https://raw.githubusercontent.com/owner/repo/main/Assets/icon.png"` |
| `defaultMinHostVersion` | string | 最低兼容宿主版本 | `"0.7.4"` |
| `tags` | string[] | 插件标签，用于分类和搜索 | `["music", "widget"]` |

### 能力提示字段

`capabilityHints` 对象声明插件提供的各类能力：

| 字段 | 类型 | 说明 |
|------|------|------|
| `capabilityHints.desktopComponents` | string[] | 桌面组件名称列表 |
| `capabilityHints.settingsSections` | string[] | 设置页节名称列表 |
| `capabilityHints.exports` | string[] | 导出的共享契约接口名称列表 |
| `capabilityHints.messageTypes` | string[] | 消息总线中使用的消息类型列表 |

### 可选字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `releaseAssetName` | string | 指定要使用的 Release Asset 名称（当 Release 中有多个 `.laapp` 时） | `"MyPlugin.1.0.0.laapp"` |

## 共享契约

如果插件导出了共享契约（供其他插件消费），还需要在注册表的 `contracts` 数组中添加契约条目：

```json
{
  "id": "LanMountainDesktop.SharedContracts.YourContract",
  "version": "1.0.0",
  "assemblyName": "LanMountainDesktop.SharedContracts.YourContract.dll",
  "downloadUrl": "https://raw.githubusercontent.com/owner/LanAirApp/main/airappmarket/contracts/YourContract/1.0.0/YourContract.dll",
  "sha256": "<64位十六进制SHA256哈希>",
  "packageSizeBytes": <文件大小字节数>
}
```

同时需要将契约 DLL 文件放置到 `airappmarket/contracts/<契约ID>/<版本号>/` 目录下。

### 计算契约 DLL 的 SHA256

```powershell
$hash = (Get-FileHash -Path "YourContract.dll" -Algorithm SHA256).Hash.ToLower()
$size = (Get-Item "YourContract.dll").Length
Write-Host "SHA256: $hash"
Write-Host "Size: $size bytes"
```

## 完整示例

以下是一个完整的收录 PR 变更示例，以 VoiceHubLanDesktop 为参考：

### 修改 `airappmarket/registry/official-plugins.json`

在 `plugins` 数组末尾添加：

```json
{
  "id": "VoiceHubLanDesktop",
  "repositoryUrl": "https://github.com/wwiinnddyy/VoiceHubLanDesktop",
  "marketManifestAssetName": "market-manifest.json",
  "projectUrl": "https://github.com/wwiinnddyy/VoiceHubLanDesktop",
  "readmeUrl": "https://raw.githubusercontent.com/wwiinnddyy/VoiceHubLanDesktop/main/README.md",
  "homepageUrl": "https://github.com/wwiinnddyy/VoiceHubLanDesktop",
  "iconUrl": "https://raw.githubusercontent.com/wwiinnddyy/VoiceHubLanDesktop/main/Assets/icon.png",
  "defaultMinHostVersion": "0.7.4",
  "tags": [
    "voicehub",
    "music",
    "playlist",
    "radio"
  ],
  "capabilityHints": {
    "desktopComponents": [
      "VoiceHubPlaylistWidget"
    ],
    "settingsSections": [
      "VoiceHubSettingsSection"
    ],
    "exports": [],
    "messageTypes": []
  }
}
```

### PR 描述

使用"插件收录申请"模板，填写：

- 插件 ID：`VoiceHubLanDesktop`
- 仓库 URL：`https://github.com/wwiinnddyy/VoiceHubLanDesktop`
- 最新版本：`v0.0.8`
- apiVersion：`4.0.0`
- 最低宿主版本：`0.7.4`
- 附上桌面组件截图和设置页截图

## 常见问题

### Q：提交 PR 后多久能审核？

维护者会在有空时尽快审核。你可以在 PR 中 @ 维护者提醒。

### Q：审核不通过怎么办？

维护者会说明不通过的原因，你可以根据反馈修改后重新提交。

### Q：插件更新版本后需要重新提交 PR 吗？

不需要。注册表中只记录仓库 URL，IndexBuilder 会自动从 GitHub API 获取最新 Release。你只需在插件仓库发布新 Release，然后触发 LanAirApp 的索引刷新工作流即可。

### Q：可以收录非 GitHub 仓库的插件吗？

目前不支持。`repositoryUrl` 必须为 `github.com` 上的仓库地址，因为 IndexBuilder 依赖 GitHub API 获取 Release 信息。

### Q：`index.json` 需要手动修改吗？

不需要。`index.json` 由 IndexBuilder 自动生成，任何手动修改都会在下次刷新时被覆盖。你只需修改 `official-plugins.json`。

### Q：如何触发索引刷新？

- 插件发布新 Release 后，可以通过 `repository_dispatch` 事件触发 LanAirApp 的刷新工作流
- 也可以在 LanAirApp 仓库的 Actions 页面手动触发 `Refresh AirAppMarket Index` 工作流
- 当 `official-plugins.json` 变更合并到 `main` 时，刷新工作流会自动触发
